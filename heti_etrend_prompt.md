\## PATCH NOTES FORMÁTUM (KÖTELEZŐ A JSON-BA)



A generált JSON-ba minden héten kötelező belefoglalni a `patch\_notes` mezőt, közvetlenül a `week\_start` után, a `days` előtt:



```json

"patch\_notes": {

&#x20; "cim": "Rövid hét-leírás — max 1 sor, tematika vagy fő jellemző",

&#x20; "feherje\_alap\_b": "A rotáló batch étel neve pontosan",

&#x20; "elvezetiek": \[

&#x20;   "Első élvezeti étel (nap + étkezés)",

&#x20;   "Második élvezeti étel (nap + étkezés)",

&#x20;   "Harmadik élvezeti étel (nap + étkezés)"

&#x20; ],

&#x20; "highlights": \[

&#x20;   "Legfontosabb egészségügyi szempont ezen a héten",

&#x20;   "Egy kiemelendő összetevő vagy tematika és miért",

&#x20;   "Meal prep tipp vagy praktikus info",

&#x20;   "Bármi ami különleges ezen a héten"

&#x20; ],

&#x20; "macro\_cel": {

&#x20;   "levi": "max 2000 kcal / nap · 140–160g fehérje",

&#x20;   "edit": "max 1550 kcal / nap · 90–110g fehérje"

&#x20; }

}

```



\*\*Szabályok:\*\*

\- `cim`: max 60 karakter, legyen informatív és motiváló, humorosan vicces

\- `feherje\_alap\_b`: pontosan az a neve amit a batch főzésnél használtál

\- `elvezetiek`: mindig 3 elem, formátum: "Étel neve (nap vacsora/ebéd)"

\- `highlights`: 3–4 elem, legyenek konkrétak (ne általánosak, viszont vigyunk bele humort)

\- `macro\_cel`: \*\*mindig ez a fix szöveg\*\*, ne változtasd a valós átlagra
